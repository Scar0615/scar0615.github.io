腳底壓力

第二章

  • 針對現有相關文獻的壓力感測系統做研究與探討

常見的腳底感測方法有兩種：將感測器做成長形地毯或是將感測系統長時間建立於鞋墊裡

做成長形地毯：這樣做不僅成本昂貴，也會因為鞋子的種類和行走速度的影響而讓感測數據出錯，進而影響感測精準度。

長時間建立於鞋墊裡：雖然不受鞋種的影響，但卻需要因為每個人不同的腳型大小而量身訂做，且長時間的放置在鞋裡，受到擠壓、溫度以及濕度的干擾，很難保證感測器不受影響而讓辨識率降低。

2.1 壓力感測設備的現況研究

使用者會將感測系統建置與鞋墊裡或是鞋子中，為了偵測使用的的日常行為動作與生活，能夠感應姿勢與壓力變化判斷人的行為。

例如圖1[9]，透過置於鞋墊內的感測器，感測使用者的行為加以分類，包括走路、站著和坐著，對於這樣的分類，它們已實現95.4%的精準度，這代表此種壓力感測設備系統已趨於能夠辨認人體動作，以此進一步在做為辨識人的依據上，有很強大的可行性。

這樣的感測系統對於醫療、運動或是年長者監控有非常優秀的幫助，不過這樣的設備幾乎都需要量身訂做，甚至需要長時間穿戴，耗損以及成本也就高了許多。

2.2 壓力辨識方法的現況研究

壓力辨識技術在接收資料後透過繁複的特徵提取使用SVM(Support Vector Machine) 進行訓練[8][9][10][11][16]，雖然訓練處理的時間較長，但其獲得的辨識結果還算不錯。

如圖3，其記錄下壓力的分布，選取其中幾個點最為特徵點，使用SVM分類識別，最後得到了97%的精準度。

2.3 研究議題

第一個議題，從現有的機器學習看來，準確率與訓練時間上要達成平衡可能還需要進一步探討與研究，許多論文裡也較避免提及收集資料與訓練所耗費的成本，腳底壓力辨識通常需要收集大量繁雜的雙腳資料，通常都需要收集腳底週期[18]、兩腳間的距離與壓力的變化，收集資料上也會較為不易，且為了使資料收集更為詳細，論文中通常會使用市面上販賣的感測系統如圖4，設備成本還會提高。

另一個議題是相關論文對於系統訓練與辨識時間上的研究較少，由於系統常常需要及時處理與辨識，處理速度顯得重要，而且SVM與CNN+RNN在機器學習的訓練時間較長，訓練與辨識時間的成本也應該要加入考慮，如此一來能夠更進一步的觀測辨識率的變化取得最佳系統辨識率。

藉由上述兩個議題可知，壓力感測系統的收集資料與特徵提取較不易、系統設備成本較昂貴。
在目前經常使用的SVM與CNN+RNN之下，雖有不錯的精準度但訓練速度所耗費的時間較為漫長，對於整體系統上的成本控制並無太多的著墨，因此這就是我們想要精進並且加強的地方。

第三章

  • 評估在這樣的狀況之下，提出能夠改進的議題。

這裡有一套快速收集與訓練的低成本腳底壓力辨識系統如圖5，使用40個感測器與10台Arduino所做成的感測系統，使用者僅需一步踏過感測平台，系統就能夠收集腳底壓力資訊轉換為特徵與圖像，接著經過機器學習訓練建立模型得出系統辨識率，將模型更新之後，就能夠去辨識不同的人。

3.3 提取特徵

電腦裡的資料庫在取得資料後，將會開始提取特徵，根據論文[5]，它們使用平均值與圖像作為一種辨識的特徵，我們將此方法拿來使用，我們針對圖像做強化處理並增加長寬與重心兩樣特徵來組合使用，並且去觀察特徵組合與原始壓力數值對於系統辨識率與訓練時間的影響。

在資料庫中，我們將一開始收集到的資料定義為原始壓力數值，每一筆資料裡有人物ID、左腳或是右腳、第幾次幀與x、y座標。接著會把各個壓力點轉換下圖8的4*10的序列圖像並經過簡易的去雜訊與強化處理。

3.4 資料儲存

在資料提取完特徵之後，我們將資料經過組合分為以下4種組合，接著將這四種組儲存進資料庫裡，當特徵組合資料需要使用時，機器學習會選擇要使用的特徵組合，進行下一步處理。
a.原始壓力數值與圖像
b.平均值與圖像
c.平均值+長寬
d.平均值+長寬+重心位置

3.5 更新模型

接著，讓使用者判斷是否要將新的資料更新模型，如果選擇更新，機器學習會將特徵組合放入機器學習，進行學習訓練然後建立模型，如選擇不更新，系統會使用舊有的模型進行辨識，辨識結果會出現特徵最為符合的人。

3.6 學習訓練並建立模型

我們會將特徵組合放入五種不同的機器學習訓練並建立模型，其中CNN使用Incepetion V3模組，RNN使用LSTM(Long Short Term Memory Network)，SVM、RF、GB的使用來自Scikit-Learn的工具，在訓練並建立完模型之後，機器學習會先得出辨識率，我們將此當作整體系統辨識率。

SVM(Support Vector Machine)為近年來相關研究與論文最常使用的機器學習，其優點在於當資料量不大時能有不錯的辨識率，缺點為時間複雜度超過1萬筆資料時，效能較為不佳。

CNN(Convolutional Neural Network)是近幾年在圖形影像辨識上較常使用的機器學習，它將資料轉為圖像做學習處理，優點在於訓練好模型就能有不錯的分類效果，相對的，運行與計算所耗費的時間與成本較大。

RNN(Recurrent Neural Network)是使用於連續性資料上的機器學習[4]，在連續資料上有不錯的成效，而腳底壓力適合使用於此，通常配合CNN來使辨識率進一步提高，不過耗能也較大。

RF(Random Forest)是針對上次訓練錯誤的結果加強權重的機器學習，對於多種類資料可以產生高準確度的分類，組合多特徵的腳底壓力資料很適合此種機器學習，訓練與辨識上也較快。

GB(Gradient Boosting)為對損失函數(loss function)處理的一種機器學習，損失函數越大，模型就越差，所以透過不斷改進損失函數來達到好的分類效果，相對於SVM，一樣的資料下，其精準度也可以比較高。

第四章

  • 對我們所提出的方法加以實驗評估其效能與成效，並從中找出最佳效益比的感測器數量與狀況。

4.1 實驗設定

這一節在探討我們所有實驗的預設值，包含硬體設備、變數設定與數據的紀錄設定，有了這些設定之後再進行四個實驗項目，項目包括腳底特徵對系統辨識率的關係、機器學習對系統辨識率的關係、感測器數量對系統辨識率的關係以及漆器學習與辨識時間的關係。

硬體與設備：

我們使用FlexiForce A201薄膜型電阻式力量感測器，誤差值約為5%~10%，尚在可接受範圍內，配合Arduino nano將資料做簡單的處理，接著把所有資訊送進PC資料庫裡面做更進一步處理。

變數設定
感測平台上40個感測器以25~29fps收集15人腳底的壓力資料，固定取25fps存於資料庫上，一個人左右腳各20次，並放在一起使用，其目的在於讓使用者之後能夠不限於使用任何一腳辨識。這樣收集，資料共有15000筆(15人*2左右腳*25fps*20次)，每筆資料(原始資料)皆有編號、使用者、左腳或右腳、40個點的壓力數值與當下的時間。我們要做四項實驗項目來得知腳底與辨識率的關係，包含腳底特徵與機器學習，及找出感測器的數量以及各個機器學習辨識資料所需的時間。

我們會將全部資料的80%經過訓練之後，拿剩餘的20%來做預測，並以混淆矩陣 (confusion matrix) 做為系統辨識率。

4.2 數據資料

數據資料
我們共使用原始資料(15000筆)、平均資料(15000/25fps=600筆)、平均加長寬(15000/25fps=600筆)、平均加長寬加重心(15000/25fps=600筆)這幾個特徵組合，CNN與RNN僅使用圖像機器學習，所以不會有長寬與重心的組合。

特徵與系統辨識率

4.3 感測器數量對系統辨識率的關係

感測器數量設為四組，分別是原始的40個、使用一半20個(包含使用中間兩列、梅花放置與減少奇數行)，以及TOP N(2≤N≤20)。
其中TOP N (2 ≤ N ≤ 20)是指將每筆感測器的數值相加後平均，取前2~20高的使用。
定義: AVG(ID,LR,X,Y) 為在第ID個人時，L或R腳時，座標在X,Y時的平均

使用40個與使用一半
